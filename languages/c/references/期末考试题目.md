# C语言试卷完整题目汇总
## 一、判断题（10题）
1. 假设有`int a[10], *p;`，则`p=&a[0]`与`p=a`等价。
2. 若有定义：`int a[2][3];`，则`a[2][10]`能够正确访问`a`数组元素`a[1][1]`。
3. 结构体定义：
```c
struct S
{
    int g; char h;
}T;
```
则可以用`T`来定义结构体变量。
4. 通过`return`语句，函数可以带回一个或一个以上的返回值。
5. `int`类型变量占用4个字节，若有定义：`int x[10]={0,2,4};`，则数组`x`在内存中所占字节数是12字节。
6. 若有`int i=10, j=0;`执行语句：
```c
if(j=0) i++;
else i--;
```
则`i`的值为11。
7. 若有定义语句：`char m[]="china";`，则一维数组的定义中，中括号`[]`省略的值为5。
8. 函数调用语句：`func(rec1,rec2+rec3,(rec4,rec5));`中，含有的实参个数是5。
9. `int a[3][4]={{1},{5},{9}};` 它的作用是将数组各行第一列的元素赋初值，其余元素值为0。
10. C语言本身不提供输入输出语句，输入和输出操作是由函数来实现的。

# ===================== 知识点总结 =====================

## 一、数据类型与标识符

| 知识点 | 核心要点 | 对应题目 |
|--------|----------|----------|
| 标识符命名规则 | 由字母、数字、下划线组成；**不能以数字开头**；**不能是关键字**；区分大小写 | 判3、单3、单4 |
| C语言关键字 | `int`、`void`、`unsigned`、`char`、`struct`、`if`、`else`、`for`、`while`、`return` 等共32个关键字，均不能作为标识符 | 单3、单4 |
| 整型数据 | `int` 占4字节（通常），`short` 2字节，`long` 4或8字节 | 判5 |
| 八进制常量 | 以 `0` 开头，如 `010` 表示十进制8 | 单8 |
| 字符与ASCII | 字符以ASCII码存储，大写字母+32=小写字母（`'A'`=65, `'a'`=97） | 单7 |

---

## 二、运算符与表达式

| 知识点 | 核心要点 | 对应题目 |
|--------|----------|----------|
| `=` vs `==` | `=` 是赋值运算符，值为被赋的值；`==` 是判等运算符。**if条件中误写 `=` 是高频错误** | 判6 |
| 复合赋值 | `a += b` 等价于 `a = a + b`；赋值表达式的值为赋值后左操作数的值 | 单5 |
| 逗号表达式 | `(a, b, c)` 从左到右求值，**整个表达式的值为最右边表达式的值**；函数调用中逗号表达式算一个实参 | 判8 |
| 三目运算符 | `条件 ? 值1 : 值2`，条件为真取值1，为假取值2 | 单7 |
| 前置++/-- | **先自增/自减，再使用**值 | 单6 |
| 后置++/-- | **先使用原值，再自增/自减** | 单6、单16、单21 |
| 短路求值 | `&&`：左侧为假则右侧不计算；`||`：左侧为真则右侧不计算 | 单6、单14 |
| 取模运算 `%` | 求余数，常用于判断奇偶（`n%2==0` 为偶数，`n%2==1` 为奇数） | 单13、单25 |

---

## 三、控制结构

| 知识点 | 核心要点 | 对应题目 |
|--------|----------|----------|
| if-else 配对 | **就近原则**：else总是与前面最近的未配对if配对，与缩进无关 | 单10、单11 |
| 空语句陷阱 | `if(x);` 末尾多余分号构成空语句，改变程序逻辑 | 单11 |
| switch-case | case后若无 `break`，会发生**穿透（fall-through）**，继续执行后续所有case | 单12 |
| for循环 | `for(初始化; 条件; 递增)` — 递增表达式在每次循环体执行完后执行 | 单14、单17 |
| while循环 | 先判断条件，再执行循环体；条件为假时退出 | 单16、单25 |
| do-while循环 | **先执行一次循环体，再判断条件**；结束条件=继续条件的否定 | 单15 |
| continue | 跳过本次循环剩余部分，**不跳过递增表达式**（for中i++仍执行） | 单17 |
| 循环条件取反 | 德摩根律：`!(A&&B)` = `!A\|\|!B`，`!(A\|\|B)` = `!A&&!B` | 单15 |

---

## 四、数组

| 知识点 | 核心要点 | 对应题目 |
|--------|----------|----------|
| 数组大小 | 元素个数 × 每个元素字节数；省略大小时由初始化列表元素个数决定 | 判5、判7、单22 |
| 下标范围 | **0 到 n-1**，越界访问属于未定义行为，结果不可预测 | 判2、单24 |
| 部分初始化 | 未显式初始化的元素**自动填充为0**（全局/静态）或垃圾值（局部） | 判9、单26 |
| 二维数组初始化 | 按行初始化，`{{1},{5},{9}}` 设置每行首列，其余为0 | 判9 |
| 字符数组与字符串 | 字符串以 `'\0'` 结尾，`"china"` 占6字节（5字符+1结束符） | 判7 |
| 数组名与指针 | 数组名在表达式中代表首元素地址（`a` 等价于 `&a[0]`） | 判1 |
| 整型数组 I/O | **不能**整体输入输出整型数组，只能逐个元素操作 | 单23、单27 |
| 字符数组 I/O | 可以用 `%s` 整体输入输出（`scanf("%s",s)` / `printf("%s",s)`） | 单23、单27 |
| 字符数组赋值 | 不能用 `=` 整体赋值（需用 `strcpy`） | 单23、单27 |

## 十、高频易错点 TOP 10

1. **`=` 与 `==` 混淆** —— `if(a=0)` 是赋值，值为0（假）；`if(a==0)` 才是判等
2. **数组下标越界** —— `a[n]` 合法下标是 0~n-1，越界是未定义行为
3. **字符串长度** —— `"china"` 占6字节（含`'\0'`），不是5字节
4. **switch 忘写 break** —— 导致case穿透，执行所有后续分支
5. **if 后多余分号** —— `if(x);` 等价于 `if(x) { }`，改变逻辑
6. **scanf 中 %c 读空格** —— `%c` 不会跳过空白，与 `%d` 行为不同
7. **后置++ 的时机** —— `y--` 先判断后自减，退出时多减了一次
8. **短路求值** —— `&&` 左侧假则右侧不执行，`||` 左侧真则右侧不执行
9. **值传递不改变实参** —— 普通变量传参，函数内修改不影响外部
10. **逗号表达式算一个实参** —— `func((a,b),c)` 是2个实参而非3个

> **考点**：结构体成员访问、printf格式符（%s和%c的区别）

### 改错1：输出1~100所有素数

**【逐处解析】**

| 序号 | 错误代码 | 正确代码 | 解析 |
|------|----------|----------|------|
| ① | `void fun(int n)` | `int fun(int n)` | 函数返回判断结果（1或0），应为int类型而非void |
| ② | `if(m<=1) k=0;` | `if(n<=1) k=0;` | 形参名为n，函数内误用了未定义的m（隐含错误，原答案未列出但实际存在于原始代码） |
| ③ | `for(i=1;i<n;i++)` | `for(i=2;i<n;i++)` | 素数的因子从2开始判断；若从1开始，任何数%1==0都会错误判定为非素数 |
| ④ | `return m;` | `return k;` | 应返回判断标志k（1是素数/0不是素数），而非返回输入值m |

> **考点**：函数返回类型、素数判断算法（从2开始试除）、形参与局部变量的使用

### 改错2：级数求圆周率

**【逐处解析】**

| 序号 | 错误代码 | 正确代码 | 解析 |
|------|----------|----------|------|
| ① | `fun()`（缺返回值类型） | `float fun()` 或 `void fun()` | 原代码省略返回值类型默认为int，不匹配实际需求 |
| ② | `while(fabs(t) <= 1e-4)` | `while(fabs(t) > 1e-4)` | 原条件反了：应在绝对值**大于**阈值时继续累加，小于等于时停止。`<=`会使循环一开始就退出（因为初始t=1.0 > 1e-4，不满足 `<=`，循环不执行） |
| ③ | `i = i + 1;` | `i = i + 2;` | 级数分母是奇数序列 1,3,5,7...，每次应+2而非+1 |
| ④-1 | `s = t;` | `s += t;`（或 `s = s + t;`） | 要求累加每一项，原代码每次覆盖而非累加 |
| ④-2 | `printf("pi=%d\n", s*4);` | `printf("pi=%f\n", s*4);` | s*4是浮点数，应用 `%f` 而非 `%d` |

> **考点**：循环条件正确性、级数累加算法、printf格式符匹配、浮点数输出

### 设计1：找最小值

```c
int fun(int x[],int n)
{
    int min = x[0];        // 假设第一个元素为最小值
    for(int i=1; i<n; i++) // 从第二个元素开始遍历
    {
        if(x[i] < min)     // 发现更小的值
            min = x[i];    // 更新最小值
    }
    return min;            // 返回最小值
}
```

**【算法思路】** 打擂台法：设第一个元素为"擂主"（min），依次与后续元素比较，遇到更小的就替换擂主，遍历完成后返回最终擂主。
> **考点**：一维数组遍历、打擂台算法（找最值）、数组名作函数参数

### 设计2：素数判断

```c
int fun(int n)
{
    if(n <= 1)              // 1及以下不是素数
        return 0;
    for(int i=2; i*i <= n; i++) // 只需试除到 sqrt(n)
    {
        if(n % i == 0)      // 能被整除说明不是素数
            return 0;
    }
    return 1;               // 没有因子，是素数
}
```

**【算法思路】** 
1. 特殊处理：≤1的数不是素数
2. 优化循环：只需试除 2 到 √n（即 `i*i <= n`），因为如果n有大于√n的因子，必然有一个小于√n的因子与之对应
3. 发现因子立刻返回0，循环结束未发现则返回1
> **考点**：素数判断算法、循环优化（试除到√n）、函数返回值设计
