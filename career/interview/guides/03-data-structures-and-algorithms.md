## 一、数据结构选型

| 结构 | 特点 | 典型场景 |
|------|------|----------|
| 数组 | 随机访问 O(1)，插入删除 O(n) | 双指针、前缀和 |
| 链表 | 插入 O(1)，查找 O(n) | 反转、合并、快慢指针 |
| 栈 | LIFO | 括号匹配、表达式、DFS |
| 队列 | FIFO | BFS、滑动窗口 |
| 哈希表 | 均摊 O(1) 查找 | 两数之和、频次统计 |
| 二叉树 | 层次/递归 | 遍历、BST、最近公共祖先 |
| 堆 | 最值 O(log n) | TopK、多路归并 |

## 二、复杂度与排序

复习 [复杂度分析](../../../fundamentals/05-algorithms/guides/01-complexity-analysis.md) 与 [排序算法](../../../fundamentals/05-algorithms/guides/02-sorting-algorithms.md)。

| 算法 | 平均时间 | 空间 | 稳定 |
|------|----------|------|------|
| 快排 | O(n log n) | O(log n) | 否 |
| 归并 | O(n log n) | O(n) | 是 |
| 堆排序 | O(n log n) | O(1) | 否 |

**快排核心**：选 pivot → 分区 → 递归左右。

## 三、经典题型与案例

### 案例 1：栈操作序列（Amazon 类题）

**题意**：给定原数组与目标数组，判断能否仅用栈的 push/pop（从左到右 push）得到目标序列，并输出操作序列。

**思路**：

```
用栈模拟：依次 push 原数组元素；
每次 push 后，若栈顶等于目标下一个元素，则 pop 并推进目标指针。
最终栈空且目标读完 → 可行。
```

**输出示例**：原 `[1,2,3,4]`，目标 `[1,2,3,4]` → `push1|pop1|push2|pop2|...`

### 案例 2：多路归并 TopK（新浪/网易类题）

**题意**：多个已排序数组，取全局前 K 小（或大）。

**思路**：小根堆存 `(值, 数组下标, 元素下标)`，初始各数组首元素入堆；弹出最小，将该数组下一元素入堆，重复 K 次。复杂度约 O(K log m)，m 为数组个数。

### 案例 3：按字节截取字符串（含中文）

**题意**：按字节截断，不劈开多字节字符（如 UTF-8 中文）。

**原则**：逐字符判断 UTF-8 首字节长度，累计字节不超过 limit；C/C++ 需注意 `char` 有符号与编码假设，面试中先说明编码前提。

### 案例 4：Singleton

**要点**：私有构造、静态实例、线程安全（双重检查锁 / 静态内部类 / C++11 `static` 局部变量）。

### 案例 5：递归与回溯

- 全排列、组合、子集：固定选择 / 不选择
- 迷宫、岛屿：DFS +  visited
- 斐波那契：裸递归 O(2^n) → 记忆化或 DP O(n)

## 四、高频模式

| 模式 | 识别信号 | 例子 |
|------|----------|------|
| 双指针 | 有序数组、链表 | 两数之和 II、去重 |
| 滑动窗口 | 连续子数组/子串最值 | 最长无重复子串 |
| 二分 | 单调性 | 搜索旋转数组 |
| 贪心 | 局部最优 → 全局 | 区间调度 |
| DP | 最优子结构 + 重叠子问题 | 背包、LCS |

## 五、刷题与现场编程

### 复习节奏

| 周期 | 内容 |
|------|------|
| 第 1～2 周 | 数组、链表、栈队列，每天 2～3 题 |
| 第 3～4 周 | 树、二分、递归 |
| 第 5～6 周 | DP、图（按目标岗位调整） |

经典题集（如「编程艺术」「100 题」类）宜**按专题拆**，配合 [fundamentals/](../../../fundamentals/) 教程，而非裸刷答案。

### 现场流程

```
1. 复述题意，举 1～2 个例子（含边界）
2. 说思路与复杂度，获确认后再写
3. 编码时分模块，变量命名清晰
4. 自测：空输入、单元素、重复、极值
5. 主动提优化（空间/时间 trade-off）
```

### 反例

- 沉默写码 20 分钟无交流
- 忽略 `null`、空数组、溢出
- 只会背题解，换条件无法调整
