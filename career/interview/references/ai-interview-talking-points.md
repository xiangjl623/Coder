# AI 相关面试话术要点

## 工程岗：我如何使用 AI

**结构**：场景 + 做法 + 验证 + 边界

> 在写单元测试和重复性样板代码时，我会用 Cursor 根据现有接口生成用例框架；合并前必须本地跑通测试并 Review diff。核心算法、鉴权和安全相关代码我自己写，不交给 AI 一次性生成。

## 工程岗：AI 不能替代什么

- 需求拆解与架构 trade-off
- 并发正确性与安全审计
- 对业务后果的判断
- 线上故障的责任承担

## LLM 应用岗：RAG 怎么讲

1. **是什么**：检索相关文档片段再生成答案  
2. **为什么**：减少幻觉、接入私有知识  
3. **注意**：分块、嵌入模型、召回、引用、权限  
4. **评测**：黄金问题集 + 人工抽检

## LLM 应用岗：Agent 怎么讲

1. **是什么**：LLM 规划 + 调用外部工具  
2. **场景**：多步任务（查库、调 API、写文件）  
3. **风险**：权限、成本、循环；需步数上限与沙箱  
4. **工程**：日志、重试、人工确认敏感操作

## 可能被问到的安全题

| 问题 | 要点 |
|------|------|
| Prompt 注入 | 用户输入改变系统指令；防护：隔离 system/user、输出过滤 |
| 数据泄露 | RAG 文档权限、日志脱敏 |
| 依赖模型 | 版本锁定、降级策略、多模型备份 |
