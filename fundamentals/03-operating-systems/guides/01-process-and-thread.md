## 进程

**进程**是操作系统进行资源分配和调度的基本单位：独立虚拟地址空间、文件描述符、权限等。

### 状态（简化）

```
新建 → 就绪 ⇄ 运行 → 终止
         ↑      ↓
         └── 阻塞（等待 I/O、锁等）
```

### 进程 vs 线程

| 维度 | 进程 | 线程 |
|------|------|------|
| 地址空间 | 独立 | 共享所属进程 |
| 创建/切换 | 开销大 | 开销小 |
| 通信 | IPC | 共享内存（需同步） |
| 崩溃 | 相对隔离 | 可能拖垮进程 |

**协程**（用户态）：更轻量，由运行时调度，非 OS 内核线程（Go、async 等）。

## 系统调用

用户程序通过 **系统调用** 请求内核服务：`fork`、`read`、`write`、`socket` 等。

```
用户态 --syscall--> 内核态 --硬件--> 完成 I/O / 调度
```

一次 syscall 有上下文切换成本 — 高频 I/O 需考虑批量、异步。

## Linux 实践

```bash
ps aux              # 进程列表
top                 # 动态监控
kill -15 <pid>      # SIGTERM 优雅结束
kill -9 <pid>       # SIGKILL 强制（慎用）
```

详见 [engineering/02-linux-and-shell/](../../../engineering/02-linux-and-shell/)。

## 面试常考

- **进程与线程区别** — 见上表
- **僵尸进程**：子进程结束父进程未 `wait` → 占 PID
- **孤儿进程**：父进程先退出，由 init 收养

## 本节小结

- 进程 = 资源容器；线程 = 执行单元
- 状态：就绪 / 运行 / 阻塞
- Linux 用 ps/top/kill 观察与管理
