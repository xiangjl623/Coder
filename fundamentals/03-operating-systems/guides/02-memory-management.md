# 内存管理

> **难度**：进阶 | **阅读时间**：约 20 分钟 | **前置**：[进程与线程](01-process-and-thread.md) | **标签**：`virtual-memory` `paging`

## 学习目标

- [ ] 解释虚拟内存的作用
- [ ] 理解分页与页表的基本思想
- [ ] 知道缺页中断与换页

## 为什么需要虚拟内存

| 问题 | 虚拟内存如何解决 |
|------|------------------|
| 进程隔离 | 各自独立地址空间 |
| 内存大于物理 RAM | 部分页放磁盘，按需换入 |
| 简化链接 | 每个进程「以为」从 0 开始 |

程序使用的是 **虚拟地址**；MMU 在访问时翻译成 **物理地址**。

## 分页

内存划分为固定大小 **页**（常见 4 KB）。页表记录：虚拟页 → 物理页框，及有效位、读写权限等。

```
虚拟地址 = 页号 + 页内偏移
         ↓ 页表
物理地址 = 页框号 + 页内偏移
```

**TLB**：页表缓存，加速地址翻译（与 [Cache](../../02-computer-architecture/guides/03-memory-hierarchy.md) 类似）。

## 缺页与换页

访问不在内存的页 → **缺页中断** → OS 从磁盘换入 → 继续执行。

磁盘慢 → **页面置换算法**（LRU、Clock 等，了解概念即可）。

## 与编程的关系

```c
int *p = malloc(n * sizeof(int));  /* 堆：进程虚拟地址空间的一部分 */
```

- **栈**：局部变量、调用链，向下增长
- **堆**：动态分配
- **代码段 / 数据段**：只读或可读写

**内存泄漏**：堆分配未释放；**野指针**：访问已释放或非法地址。

## 本节小结

- 虚拟内存 = 隔离 + 扩展 + 简化编程模型
- 分页 + 页表 + TLB
- 缺页换页是性能敏感路径
