# 指令系统与 CPU

> **难度**：入门 | **阅读时间**：约 18 分钟 | **前置**：[存储器层次结构](03-memory-hierarchy.md) | **标签**：`cpu` `isa` `pipeline`

## 学习目标

- [ ] 理解取指、译码、执行的基本周期
- [ ] 知道流水线提高吞吐、分支可能带来冒险
- [ ] 区分时钟频率与 IPC 对性能的影响

## 指令集架构（ISA）

ISA 是软硬件接口：CPU 支持的指令集合（如 x86-64、ARM、RISC-V）。

| 风格 | 特点 |
|------|------|
| CISC | 指令功能丰富（x86） |
| RISC | 指令简单、固定长度（ARM、RISC-V） |

编译器生成的是 **机器码**（ISA 指令的二进制编码）。

## 指令周期

```
取指 (Fetch) → 译码 (Decode) → 执行 (Execute) → 访存 → 写回
```

循环执行；现代 CPU 同一时刻有多条指令在不同阶段（**流水线**）。

## 流水线与分支

理想流水线：每个时钟完成一条指令阶段，吞吐接近 1 IPC。

**分支指令**（if、循环跳转）可能导致：

- 流水线清空（预测失败）
- 需 **分支预测** 硬件猜测方向

**对程序员**：可预测分支、减少深层不可预测分支有助于性能（微优化，通常次于算法）。

## 时钟与性能

```
CPU 时间 ≈ 指令数 × CPI × 时钟周期
```

- 提高主频 → 缩短周期
- 降低 CPI → 更好流水线、更少 stall
- 减少指令数 → 编译优化、更好算法

**反例**：只比 GHz 高低判断 CPU 强弱。

## 多核

多核 = 多个执行单元共享或独立 Cache/内存。并行需 [03 操作系统：进程与线程](../../03-operating-systems/guides/01-process-and-thread.md) 与同步机制配合。

## 本节小结

- CPU 反复取指执行；流水线提高并行度
- 性能 = 指令数 × CPI × 周期
- ISA 是编译器与芯片的契约
