# 存储器层次结构

> **难度**：入门 | **阅读时间**：约 18 分钟 | **前置**：[数据的表示与运算](02-data-representation.md) | **标签**：`cache` `memory`

## 学习目标

- [ ] 说出寄存器 → Cache → 内存 → 磁盘的层次与速度差异
- [ ] 理解局部性原理
- [ ] 知道数组顺序访问通常比随机访问快的原因

## 层次金字塔

```
        寄存器     ← 最快、最小、CPU 内部
          ↓
        Cache (L1/L2/L3)
          ↓
        主存 RAM
          ↓
        磁盘 / SSD   ← 最慢、最大、持久
```

**数量级直觉**（相对，非精确）：

| 层次 | 访问时间量级 |
|------|--------------|
| L1 Cache | ~1 ns |
| 内存 | ~100 ns |
| SSD | ~10–100 μs |
| 机械硬盘 | ~ms |

## 局部性原理

- **时间局部性**：刚访问的数据可能再次访问（循环变量）
- **空间局部性**：相邻地址可能接着访问（数组顺序遍历）

**反例**：链表跳跃访问、大哈希表随机 probe → Cache 命中率低。

```c
/* 友好：顺序访问 */
for (int i = 0; i < n; i++)
    sum += a[i];

/* 不友好：列优先遍历二维数组（语言/布局相关） */
```

## Cache 与行（Cache Line）

内存按 **Cache Line**（常见 64 字节）加载。一次 miss 载入一整行 — 顺序访问摊薄成本。

**伪共享**（进阶）：多线程改同一 Cache Line 内不同变量 → 性能下降。

## 与 OS 虚拟内存的关系

进程看到连续虚拟地址；OS + MMU 映射到物理页。缺页时从磁盘换入 — 见 [03 操作系统：内存管理](../../03-operating-systems/guides/02-memory-management.md)。

## 本节小结

- 越靠近 CPU 越快越贵
- 写好循环与数据布局 ≈ 免费性能
- 磁盘 I/O 是多数应用终极瓶颈之一
