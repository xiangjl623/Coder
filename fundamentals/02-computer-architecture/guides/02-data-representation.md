# 数据的表示与运算

> **难度**：入门 | **阅读时间**：约 20 分钟 | **前置**：[计算机系统层次](01-computer-system-layers.md) | **标签**：`binary` `two-complement`

## 学习目标

- [ ] 能在二进制、十六进制与十进制间转换
- [ ] 理解有符号整数的补码表示
- [ ] 知道浮点数存储是近似值（面试防坑）

## 进制

| 进制 | 用途 |
|------|------|
| 二进制 | CPU 内部 |
| 十六进制 | 调试、地址（1 位 hex = 4 位 binary） |
| 十进制 | 人类阅读 |

**示例**：`0xFF` = 255 = `11111111`₂

## 有符号整数：补码

n 位补码表示范围约 **[-2^(n-1), 2^(n-1)-1]**。

- 正数：原码即二进制
- 负数：按位取反 + 1

**为何用补码**：加减法统一硬件电路；0 的表示唯一。

```c
/* 8 位 char 上 -1 的补码常为 0xFF */
signed char a = -1;
unsigned char b = 255;
/* a 与 b 位模式相同，解释不同 — C 语言期末常考 */
```

## 整数溢出

有符号溢出 → 未定义行为（C）；无符号溢出 → 模 2^n。

**面试点**：`INT_MAX + 1` 可能变负数；比较混用 signed/unsigned 时 unsigned 会「变大」。

## 浮点数（概览）

IEEE 754：符号位 + 指数 + 尾数。**不能精确表示所有十进制小数**（如 0.1）。

```c
if (0.1 + 0.2 == 0.3)  /* 常为 false */
```

金额计算宜用整数分（`long` 存分）或 `decimal` 类型（其他语言）。

## 字符与编码

ASCII 基础；中文常用 UTF-8 变长编码（与 [面试：按字节截字符串](../../../career/interview/guides/03-data-structures-and-algorithms.md) 相关）。

## 本节小结

- 补码是有符号整数主流表示
- 浮点比较用误差范围，勿直接 `==`
- 混用 signed/unsigned 是常见 bug 源

## 练习

1. 计算 8 位补码 `-5` 的二进制
2. 解释 `char c = 200;` 在有符号 char 上可能输出负数
