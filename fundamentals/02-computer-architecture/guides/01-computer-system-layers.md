# 计算机系统层次

> **难度**：入门 | **阅读时间**：约 15 分钟 | **前置**：无 | **标签**：`architecture` `abstraction`

## 学习目标

- [ ] 理解从硬件到应用的层次抽象
- [ ] 说清编译、链接、加载在程序运行中的位置
- [ ] 知道性能问题可能在哪一层定位

## 层次结构

```
应用（浏览器、游戏、数据库）
    ↓ 系统调用 / API
操作系统（进程、内存、文件、驱动）
    ↓ 指令集架构 ISA
硬件（CPU、内存、I/O、总线）
```

程序员日常在**应用层**写代码，通过**操作系统**使用 CPU 与磁盘；编译器把高级语言翻译成 **ISA 指令**（如 x86-64、ARM）。

## 抽象的价值

| 抽象 | 对程序员隐藏 |
|------|--------------|
| 进程 | 物理 CPU 如何在多任务间切换 |
| 虚拟内存 | 物理地址与页表 |
| 文件 | 磁盘扇区布局 |
| 指令集 | 微码、流水线细节 |

**原则**：下层更快、更难改；上层更灵活、更易移植。

## 程序如何运行（简版）

```
源代码 (.c) --编译--> 目标文件 (.o) --链接--> 可执行文件
    --加载--> 内存中的进程 --CPU 取指执行--> 结果 / I/O
```

与 [languages/c/](../../../languages/c/) 对照：编译链接见环境搭建章节。

## 性能直觉

- **CPU  bound**：计算密集（加密、排序）
- **Memory bound**：缓存未命中、随机访问大数组
- **I/O bound**：磁盘、网络等待

优化前先 profiling，避免在错误层次优化（例如在应用层狂改算法，瓶颈却在磁盘）。

## 本节小结

- 计算机 = 硬件 + OS + 应用，层层抽象
- 程序运行经历编译、链接、加载、取指执行
- 性能问题需定位层次
