# 复杂度分析

> **难度**：入门 | **阅读时间**：约 15 分钟 | **前置**：[数组与链表](01-arrays-and-linked-lists.md) | **标签**：`big-o` `complexity`

## 学习目标

- [ ] 理解时间复杂度描述「规模增长时操作次数的趋势」
- [ ] 能识别 O(1)、O(n)、O(n²)、O(log n) 常见代码模式
- [ ] 区分最坏、平均、最好情况（以查找为例）

## 为什么需要复杂度

同一问题可有多种算法。当数据规模 n 增大时，我们需要比较**增长趋势**，而非绝对运行时间（与机器、语言有关）。

## 大 O 表示法

表示上界：当 n 足够大时，运行时间不超过 c·f(n)。

| 复杂度 | 名称 | 典型例子 |
|--------|------|----------|
| O(1) | 常数 | 数组下标访问 |
| O(log n) | 对数 | 二分查找 |
| O(n) | 线性 | 遍历数组 |
| O(n log n) | 线性对数 | 快排、归并（平均） |
| O(n²) | 平方 | 双重循环、冒泡排序 |

```c
/* O(n) */
for (int i = 0; i < n; i++) { /* ... */ }

/* O(n²) */
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++) { /* ... */ }
```

## 空间复杂度

算法额外使用的内存随 n 的增长趋势。原地排序空间 O(1)；递归深度为 n 时栈空间 O(n)。

## 本节小结

- 大 O 忽略常数和低阶项，关注主导项
- 嵌套循环层数往往是复杂度的重要线索
- C 语言期末程序阅读题常隐含复杂度判断

## 练习

1. 判断下列复杂度：单层循环 n 次；循环 i 从 1 到 n 每次 j 到 i；while (n>1) n/=2
2. 说明为什么二分查找是 O(log n)
