# 数组与链表

> **难度**：入门 | **阅读时间**：约 20 分钟 | **前置**：[C 语言数组](../../languages/c/guides/02-core-syntax/08-数组与字符串：批量数据处理.md) | **标签**：`array` `linked-list`

## 学习目标

- [ ] 理解数组的连续存储与 O(1) 随机访问
- [ ] 理解链表的节点结构与 O(n) 访问
- [ ] 能在 C 语言中实现单向链表的基本操作

## 数组：连续内存

数组元素在内存中**连续存放**，通过下标可直接计算地址：

```
地址 = 基址 + 下标 × 元素大小
```

| 操作 | 时间复杂度 |
|------|------------|
| 按下标访问 | O(1) |
| 末尾插入（定长数组） | 不可行或 O(1) 若未满 |
| 中间插入 | O(n) 需移动元素 |

```c
int arr[5] = {1, 2, 3, 4, 5};
printf("%d\n", arr[2]);  /* O(1) 访问 */
```

## 链表：离散节点 + 指针

每个节点包含**数据**和**指向下一个节点的指针**：

```c
typedef struct Node {
    int data;
    struct Node *next;
} Node;
```

| 操作 | 时间复杂度 |
|------|------------|
| 按值查找 | O(n) |
| 头部插入 | O(1) |
| 中间/尾部插入 | O(n) 需先找到位置 |

## 数组 vs 链表

| 对比项 | 数组 | 链表 |
|--------|------|------|
| 内存 | 连续，可能浪费或溢出 | 按需分配，有指针开销 |
| 访问速度 | 快，缓存友好 | 慢，需遍历 |
| 插入删除 | 中间操作代价高 | 已知节点时插入 O(1) |
| 适用场景 | 固定大小、频繁随机访问 | 大小不确定、频繁插入删除 |

## 常见误区

- **误区**：链表一定比数组快。  
  **正确**：仅当插入删除频繁且不需要随机访问时，链表更有优势。
- **误区**：忘记释放链表节点导致内存泄漏。  
  **正确**：遍历 `free` 每个节点，或维护尾指针简化操作。

## 本节小结

- 数组适合已知规模、随机访问；链表适合动态增删
- C 语言期末常考单向链表的创建、遍历、插入、删除
- 复杂度分析是选型依据
