# 传输层：TCP 与 UDP

> **难度**：进阶 | **阅读时间**：约 22 分钟 | **前置**：[链路与 IP](02-link-layer-and-ip.md) | **标签**：`tcp` `udp` `handshake`

## 学习目标

- [ ] 对比 TCP 与 UDP 的特性与场景
- [ ] 描述 TCP 三次握手与四次挥手
- [ ] 理解可靠传输与流量控制的基本思想

## TCP vs UDP

| | TCP | UDP |
|---|-----|-----|
| 连接 | 面向连接 | 无连接 |
| 可靠 | 确认、重传、有序 | 不保证 |
| 开销 | 头部较大、状态多 | 轻量 |
| 场景 | HTTP、文件、邮件 | DNS、视频、游戏、VoIP |

## TCP 三次握手

建立连接前，双方同步初始序列号：

```
客户端 --SYN--> 服务端
客户端 <--SYN+ACK-- 服务端
客户端 --ACK--> 服务端
```

**为何三次**：防止旧连接请求突然到达导致误建立；双方确认收发能力。

## TCP 四次挥手

关闭连接需双方均关闭写方向：

```
A --FIN--> B
A <--ACK-- B
A <--FIN-- B   （B 也关闭）
A --ACK--> B
```

**TIME_WAIT**：主动关闭方等待 2MSL，确保迟到的包不影响新连接。

## 可靠性与滑动窗口

- **序号 / 确认号**：按序确认，丢包重传
- **滑动窗口**：流量控制，避免发太快淹没接收方
- **拥塞控制**：慢启动、拥塞避免（了解概念）

## UDP 适用场景

- 可容忍丢包、低延迟优先（直播、游戏位置同步）
- 请求-响应短报文（DNS 查询）
- 应用层自己实现可靠性（QUIC 基于 UDP，HTTP/3）

## 面试答题结构

1. **是什么**：TCP 面向连接的字节流服务
2. **原理**：握手、序号、ACK、重传
3. **注意**：挥手 TIME_WAIT；UDP 无连接但可 broadcast/multicast

## 本节小结

- TCP = 可靠有序；UDP = 快、简单
- 三次握手建立、四次挥手关闭
- 选协议看延迟、丢包容忍度
