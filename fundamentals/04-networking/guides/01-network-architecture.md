# 网络体系结构

> **难度**：入门 | **阅读时间**：约 18 分钟 | **前置**：[03 操作系统](../03-operating-systems/) | **标签**：`osi` `tcp-ip`

## 学习目标

- [ ] 对照 OSI 七层与 TCP/IP 四层模型
- [ ] 理解封装与解封装过程
- [ ] 知道各层典型协议与设备

## 为什么分层

每层只关心与相邻层的接口；修改一层不影响其他层（理想情况）。便于教学、实现与排障。

## OSI 与 TCP/IP

| OSI（7 层） | TCP/IP（4 层） | 协议示例 | 设备 |
|-------------|----------------|----------|------|
| 应用 | 应用 | HTTP、DNS、SMTP | — |
| 表示 | （并入应用） | TLS、JSON | — |
| 会话 | （并入应用） | Cookie | — |
| 传输 | 传输 | TCP、UDP | — |
| 网络 | 网络 | IP、ICMP | 路由器 |
| 数据链路 | 网络接口 | 以太网、Wi-Fi | 交换机 |
| 物理 | 网络接口 | 电缆、光纤 | 集线器 |

**记忆**：面试常画 **TCP/IP 四层** 即可。

## 封装过程

```
HTTP 报文
  ↓ 加 TCP 头
TCP 段
  ↓ 加 IP 头
IP 包
  ↓ 加帧头尾
以太网帧 → 物理信号
```

接收端自下而上解封装，每层处理本层头并上交。

## 端到端原则

**传输层** TCP/UDP 在主机之间端到端；**网络层** IP 负责逐跳路由到目标主机。

## 本节小结

- 分层 = 职责分离
- TCP/IP：应用 / 传输 / 网络 / 链路
- 排障时可按层定位（ping 通？DNS 解析？HTTP 状态码？）
