# 链路与 IP

> **难度**：入门 | **阅读时间**：约 18 分钟 | **前置**：[网络体系结构](01-network-architecture.md) | **标签**：`ip` `ethernet`

## 学习目标

- [ ] 理解 MAC 地址与 IP 地址的分工
- [ ] 知道 ARP 的作用
- [ ] 区分 IPv4 地址与端口

## 数据链路层

同一局域网内，帧通过 **MAC 地址** 送达。交换机根据 MAC 表转发。

## 网络层：IP

**IP 地址**标识主机（网络 + 主机号）。路由器根据路由表转发 **包**。

| 类型 | 示例 | 说明 |
|------|------|------|
| IPv4 | 192.168.1.10 | 32 位，约 43 亿地址 |
| 私有地址 | 10.x、172.16–31.x、192.168.x | 内网，NAT 出公网 |
| 回环 | 127.0.0.1 | 本机 |

**子网掩码 / CIDR**：划分网络位与主机位，如 `192.168.1.0/24`。

## ARP

局域网内「IP → MAC」解析：**ARP 请求**广播，目标主机 ARP 应答。

跨网段通信：包先发到 **默认网关**（路由器 MAC）。

## IP 与端口

- **IP**：找到哪台主机
- **端口**（0–65535）：找到主机上哪个程序

```
192.168.1.10:443  →  该机的 HTTPS 服务
```

**well-known 端口**：80 HTTP、443 HTTPS、22 SSH、3306 MySQL。

## ICMP 与 ping

`ping` 使用 ICMP 回显请求/应答，测试 **IP 层可达性**（不保证应用层正常）。

```bash
ping 8.8.8.8
```

## 路由（概念）

主机路由表决定：本网直连 / 经网关转发。`traceroute` 观察路径跳数。

## 本节小结

- 局域网 MAC，跨网 IP + 路由
- ARP 解析 IP→MAC
- IP:端口 定位服务
